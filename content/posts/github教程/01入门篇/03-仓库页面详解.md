# 03 · 仓库页面详解

> **这篇的目标**：学会看仓库页面——认识所有标签栏、了解右侧面板信息、看懂文件浏览区、创建你的第一个仓库。

---

## 1. 什么是仓库页面

当你进入一个仓库（如 github.com/facebook/react），看到的就是**仓库页面**。它是 GitHub 最核心的页面类型，所有代码、Issue、PR、讨论都在这里管理。掌握它的布局就等于掌握了 GitHub 80% 的操作。

---

## 2. 仓库顶部信息区

### 2.1 仓库名和路径

页面左上角的面包屑导航显示：**组织/用户名 → 仓库名**，如 `facebook/react`。

旁边标注仓库类型：
- **Public**（公开）—— 所有人都能看到
- **Private**（私有）—— 只有你和协作者能看到

### 2.2 仓库统计按钮（右侧）

位于仓库名称右侧的按钮行：

| 按钮 | 作用 |
|------|------|
| **Watch** | 设置通知：Participating / All / Ignore |
| **Fork** | 复制仓库到你的账号 |
| **Star** | 收藏/点赞，Star 数是项目流行度的重要指标 |
| **Sponsor** | 赞助该项目（如果项目开启了赞助）|
| **Add to list** | 将仓库归类到你的 Star lists |

每个按钮旁边显示数字（Watch 人数、Fork 数、Star 数）。

---

## 3. 仓库标签栏（Tabs）

仓库页面中部有一行标签栏，是切换功能区的核心导航：

| 标签 | 说明 |
|------|------|
| **Code** | 代码浏览、文件管理、README（默认视图）|
| **Issues** | 问题追踪、Bug 报告、功能请求 |
| **Pull Requests** | 代码审查与合并 |
| **Actions** | CI/CD 自动化工作流 |
| **Projects** | 项目看板管理 |
| **More**（更多）| 展开下拉菜单 |

**More 菜单包含**：

| 子页面 | 说明 |
|--------|------|
| **Wiki** | 维基文档 |
| **Security** | 安全分析与 Dependabot |
| **Insights** | 数据统计与贡献图 |
| **Settings** | 仓库设置（需要管理员权限）|

---

## 4. 右侧面板（About）

仓库页面的右侧有一个信息面板，显示项目的 **About（简介）**：

| 区域 | 内容 |
|------|------|
| **Description** | 仓库的简短描述 |
| **Website** | 项目官网链接 |
| **License** | 开源许可证（如 MIT）|
| **Code of Conduct** | 行为准则 |
| **Contributing** | 贡献指南 |
| **Security Policy** | 安全策略 |
| **Custom properties** | 仓库自定义属性 |
| **Branches / Tags / Activity** | 分支数、标签数、活跃度 |

---

## 5. Code 标签页主内容区

Code 标签页是仓库页面默认显示的视图。

### 5.1 文件浏览器顶部工具栏

| 元素 | 作用 |
|------|------|
| **分支选择器** | 切换分支，默认 main |
| **Tags** | 查看版本标签 |
| **Go to file** | 按文件名快速跳转（快捷键 t）|
| **Add file** | 直接在 GitHub 上创建或上传文件 |
| **Code 按钮** | 展开：Clone URL（HTTPS / SSH）、下载 ZIP |

### 5.2 最新提交信息

工具栏下方显示最近一次提交的信息：提交者头像+名字、提交信息、提交 SHA（短编号）、时间、commit hash（点击可查看详情）、History（查看提交历史）。

### 5.3 文件和文件夹列表

以表格形式展示当前目录的文件和文件夹：

| 列 | 内容 |
|------|------|
| **Name** | 文件名/文件夹名 |
| **Last commit message** | 最后一次提交的信息 |
| **Last commit date** | 最后一次提交的时间 |

- 点击文件夹进入子目录
- 点击文件查看文件内容

### 5.4 README

文件列表下方是 **README.md** 的渲染内容。这是仓库的名片，通常包含：项目介绍和用途、安装和使用方法、技术栈说明、贡献方式。

> 进入一个新仓库，**先看 README** 是最快的了解方式。

---

## 6. 创建你的第一个仓库

### 方法一：顶部导航创建

点击右上角 ➕ **Create new...** → **New repository**。

### 方法二：Dashboard 创建

在 Dashboard 左侧的 Top Repositories 底部，有 **Create new** 按钮。

### 创建步骤

| 字段 | 说明 | 建议 |
|------|------|------|
| **Repository name** | 仓库名称 | 简短有意义（如 my-first-repo）|
| **Description** | 可选描述 | 一句话说明用途 |
| **Public / Private** | 公开或私有 | 初学者选 Public |
| **Initialize with README** | 自动创建 README | 建议勾选 |
| **Add .gitignore** | 忽略规则 | 根据主要语言选择 |
| **Choose a license** | 开源许可证 | 建议 MIT |

点击 **Create repository** 完成。

✅ 成功标志：看到新仓库页面，里面有 README 文件。
