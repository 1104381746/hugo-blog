# CC Switch 供应商管理完全指南 —— 从入门到精通

> 适用版本：CC Switch v3.16.0+ | 最后更新：2026-06-05

供应商管理是 CC Switch 的核心功能。这篇博客带你摸透从添加、切换、编辑、排序到用量查询的每一个环节，包括 Claude Desktop 的特殊用法。

## 添加供应商

点击主界面右上角的 **+** 按钮，你会看到两个 Tab：
- **应用专属供应商**：仅用于当前选中的应用
- **统一供应商**：跨应用共享配置

### 使用预设（推荐）

CC Switch 内置了丰富的预设模板，覆盖主流供应商：

| 应用                           | 预设示例                                                                            |
| ---------------------------- | ------------------------------------------------------------------------------- |
| Claude                       | DeepSeek、智谱 GLM、Kimi、MiniMax、PackyCode、SiliconFlow、OpenRouter、Nvidia、小米 MiMo 等  |
| Codex                        | 原生 Responses 协议（OpenAI 官方、Azure）、Chat Completions 协议（DeepSeek、Kimi、GLM），需自动路由映射 |
| Gemini                       | Google 官方、PackyCode、Cubence、OpenRouter 等                                        |
| OpenCode / OpenClaw / Hermes | 类似 Claude 的丰富预设列表                                                               |

选择预设后，端点地址自动填充，只需填写 API Key 即可。

### 自定义配置

预设列表中没有你的供应商时，选择「自定义」手动编辑 JSON 配置。以 Claude 为例：

```json
{
  "env": {
    "ANTHROPIC_API_KEY": "your-api-key",
    "ANTHROPIC_BASE_URL": "https://api.example.com"
  }
}
```

### 统一供应商

统一供应商可以跨 Claude Code / Codex / Gemini 共享配置，特别适合一次配置多应用使用的中转服务。创建时勾选要同步的应用，修改后自动同步到所有关联应用。

### 自动获取模型

添加或编辑供应商时，点击模型输入框旁的「获取模型」按钮（下载图标），CC Switch 会自动调用供应商的 `/v1/models` 端点拉取可用模型列表。支持按类别分组选择，省去手动粘贴模型 ID 的麻烦。

### 高级选项

- **API 格式（仅 Claude）**：支持 Anthropic Messages（默认）、OpenAI Chat Completions、OpenAI Responses API 三种格式，由代理自动转换
- **完整 URL 端点模式**：适用于需要非标准路径的供应商（勾选后 `base_url` 被视为完整上游端点，不再拼接路径）
- **Codex 本地路由映射**：让只支持 Chat Completions 协议的供应商（如 DeepSeek、Kimi）能配合 Codex 使用，需要开启本地代理
- **Codex 1M 上下文窗口**：启用后设置 100 万 token 上下文长度

### Codex OAuth 反向代理

v3.13.0 起的新功能，让你用 **ChatGPT 账号**在 Claude Code 中使用 Codex 服务。通过 Device Code 流程登录 ChatGPT，CC Switch 自动管理 Token 刷新和配额展示。

> ⚠️ 此功能使用逆向 OAuth 流程，存在违反 OpenAI 服务条款和账号风险，使用前请仔细阅读软件内的风险提示。

## 切换供应商

### 主界面切换

点击目标供应商卡片的「启用」按钮，CC Switch 自动更新配置文件。Claude/Gemini 即时生效，Codex 需重启终端。

### 托盘快速切换

右键系统托盘图标 → 悬停在应用子菜单（如 "Claude · 当前供应商"） → 点击目标供应商。完全不需要打开主界面。

托盘菜单从 v3.13.0 起重构为**按应用分组的分级子菜单**，子菜单标题直接显示当前激活的供应商和可用用量摘要。

### 生效方式一览

| 应用 | 修改的文件 |
|------|-----------|
| Claude | `~/.claude/settings.json` |
| Codex | `~/.codex/auth.json` + `~/.codex/config.toml` |
| Gemini | `~/.gemini/.env` + `~/.gemini/settings.json` |

切换失败时，检查配置文件是否被其他程序锁定、权限是否足够、JSON 格式是否有误。

## 编辑供应商

鼠标悬停卡片 → 点击「编辑」按钮。可修改的内容包括：

- **基本信息**：名称、备注、网站链接、图标和颜色（图标库支持搜索，颜色可自定义）
- **配置信息**：API Key、端点地址、JSON 配置（编辑器带语法高亮和格式校验）
- **端点管理**：添加多个端点用于测速和故障转移

### 回填机制

编辑当前启用的供应商时，CC Switch 会从 live 配置文件读取最新内容。如果你在 CLI 工具中手动改过配置，这些修改会被同步回来，不会丢失。

### 快捷开关（Claude）

编辑 Claude 供应商时，JSON 编辑器上方提供一组 **快捷开关**，勾选或取消后配置变更会实时反映在下方的 JSON 编辑器中：

| 开关 | 效果 | 对应的配置变更 |
|------|------|---------------|
| **隐藏署名** | 清除 git 提交和 PR 中的署名元数据，适合匿名贡献场景 | 设置 `attribution: {commit: "", pr: ""}` |
| **启用 Teammates** | 开启 Claude Code 的 Agent 团队协作功能，允许多 Agent 协同工作 | 设置 `env.CLAUDE_CODE_EXPERIMENTAL_AGENT_TEAMS = "1"` |
| **启用工具搜索** | 启用 Claude 的工具搜索（Tool Search）能力，在调用工具时自动检索匹配的工具 | 设置 `env.ENABLE_TOOL_SEARCH = "true"` |
| **最大强度思考** | 将 Claude 的 reasoning effort 设为 max 级别，适合处理复杂推理任务 | 设置 `env.CLAUDE_CODE_EFFORT_LEVEL = "max"` |
| **禁用自动更新** | 阻止 Claude Code 自动升级版本，适合需要固定版本的生产环境 | 设置 `env.DISABLE_AUTOUPDATER = "1"` |

取消勾选开关时，对应的配置项会被**完全移除**（而不是设为 false），避免残留空配置影响运行。

#### 使用场景举例

| 场景 | 建议开关 |
|------|---------|
| 在公司仓库提交代码，不想暴露个人署名 | 打开 **隐藏署名** |
| 需要多个 Agent 分工处理复杂任务 | 打开 **启用 Teammates** |
| Codex 迁移到 Claude Code 初期，习惯 Codex 的工具查找方式 | 打开 **启用工具搜索** |
| 分析复杂代码库、重构核心逻辑 | 打开 **最大强度思考** |
| 生产环境 CI/CD 中使用 Claude，不希望自动更新引入不稳定 | 打开 **禁用自动更新** |

#### 写入通用配置

快捷开关区域还有一个 **写入通用配置** 复选框：

- **勾选后**：当前供应商编辑的内容会合并到一个全局配置片段中，其他 Claude 供应商也能共享这部分配置
- **自定义共享配置**：点击旁边的 **编辑通用配置** 按钮，可以修改这个全局共享片段的内容

适合的场景：多个供应商使用相同的环境变量或参数，统一维护避免重复。

## 排序与复制

- **拖拽排序**：按住卡片左侧 ≡ 手柄上下拖动，排序影响故障转移队列的默认顺序
- **复制供应商**：点击「复制」创建完整副本（名称加 `copy` 后缀），适合基于现有配置创建变体
- **删除**：点击「删除」按钮并确认。统一供应商删除后，关联应用配置同步删除

## 用量查询

### 自动查询（无需配置）

以下类型开箱即用：Claude / Codex / Gemini 官方订阅、GitHub Copilot、Codex OAuth 反向代理。卡片底部自动显示使用百分比 + 重置倒计时，颜色随使用率变化（< 70% 绿 / 70–89% 橙 / ≥ 90% 红）。

### 手动启用（内置模板）

对非官方订阅类供应商（如 Token Plan、第三方余额），需手动开启「用量查询」开关并选择模板。内置模板覆盖：Kimi / 智谱 GLM / MiniMax 的套餐配额，DeepSeek / StepFun / SiliconFlow / OpenRouter / Novita AI 的余额查询。

### 自定义脚本

未被内置模板覆盖的供应商，可用 JavaScript 编写自定义查询脚本。CC Switch 提供通用模板和 New API 模板作为起点，支持模板变量替换（`{{apiKey}}`、`{{baseUrl}}` 等）。

## Claude Desktop 专项

Claude Desktop 面板独立管理 Claude 桌面应用的第三方供应商配置。

### 两种工作模式

- **直连模式**：供应商本身提供 Anthropic Messages API 且模型 ID 为 Claude Desktop 可识别的三档角色（`claude-sonnet-*` / `claude-opus-*` / `claude-haiku-*`）
- **模型映射模式**：模型名非三档角色 ID（包括旧式 Claude ID 和 DeepSeek、Kimi 等非 Claude 模型），通过 CC Switch 本地网关映射为 Sonnet / Opus / Haiku 路由

### 从 Claude Code 导入

第一次进入 Claude Desktop 面板时，可以直接将 Claude Code 中已有的供应商一键导入，省去逐个配置的时间。

### 配置文件位置

- **macOS**：`~/Library/Application Support/Claude-3p/`
- **Windows**：`%LOCALAPPDATA%\Claude-3p`

> **成功标志**：供应商卡片显示蓝色边框（启用）或绿色边框（代理活跃），底部显示用量数据。
